1. OOP Basics — Very Simple

Class

English: A class is a blueprint. It describes what data and functions an object will have.

بالعربي: الصنف هو قالب أو مخطط. يحدد البيانات والدوال التي سيملكها الكائن.
class Student {
private:
    int id;
public:
    void setId(int x) { id = x; }
};
Memory: class = design, object = real thing.

Object

English: An object is a real instance created from a class.

بالعربي: الكائن هو نسخة حقيقية تم إنشاؤها من الصنف.
Student s1;   // object from Student class
Student s2;   // another object
Each object has its own data. كل كائن له بياناته الخاصة.

Encapsulation

English: Put data and functions together inside one class.

بالعربي: جمع البيانات والدوال داخل صنف واحد.
class BankAccount {
private:
    double balance;
public:
    void deposit(double amount) {
        balance += amount;
    }
};

Data Hiding

English: Hide data using private, then control access using public functions.

بالعربي: نخفي البيانات باستخدام private ونسمح بالوصول لها عن طريق دوال public.
Trap: obj.privateVar is illegal outside the class.

Inheritance

English: A derived class receives features from a base class.

بالعربي: الصنف الابن يرث خصائص ودوال من الصنف الأب.
class Car : public Vehicle {
};

Polymorphism

English: Same name/interface, different behavior.

بالعربي: نفس الاسم أو الواجهة، لكن سلوك مختلف حسب الكائن أو النوع.

Examples: function overloading, operator overloading, virtual functions.

2. Classes and Objects

General Syntax

class ClassName {
private:
    // attributes / data members

public:
    // member functions
};
شرح: داخل الصنف نضع البيانات والدوال. غالباً البيانات تكون private والدوال تكون public. لا تنسَ الفاصلة المنقوطة بعد قوس نهاية الصنف };.

Full Simple Example

#include <iostream>
#include <string>
using namespace std;

class Student {
private:
    int number;
    string name;

public:
    void setData(int n, string s) {
        number = n;
        name = s;
    }

    void printData() {
        cout << "Number: " << number << endl;
        cout << "Name: " << name << endl;
    }
};

int main() {
    Student st;
    st.setData(1, "Ali");
    st.printData();
    return 0;
}
LineMeaningArabic
class StudentDefine a new class called Student.تعريف صنف جديد اسمه Student.
privateOnly class functions can access these variables.فقط دوال الصنف تستطيع الوصول لهذه المتغيرات.
publicCan be called from main.يمكن استدعاؤها من main.
Student st;Create object.إنشاء كائن.
st.setData(...)Call member function using dot.استدعاء دالة باستخدام النقطة.
Common errors:
  • Forgetting ; after class.
  • Trying to access private data directly.
  • Writing student st(); when you mean an object; that can be interpreted as a function declaration.
أخطاء شائعة: نسيان الفاصلة المنقوطة بعد الصنف، محاولة الوصول إلى private مباشرة، أو كتابة object بطريقة خاطئة.

3. Constructors and Destructors

Constructor

A constructor runs automatically when an object is created. It initializes data.

الـ Constructor يعمل تلقائياً عند إنشاء الكائن. وظيفته إعطاء قيم ابتدائية للبيانات.
class Counter {
private:
    int count;

public:
    Counter() {      // constructor
        count = 0;
    }
};

Constructor Overloading

You can have more than one constructor if parameters are different.

يمكن أن يكون للصنف أكثر من Constructor إذا اختلفت المعاملات.
class Student {
private:
    int id;

public:
    Student() {
        id = 0;
    }

    Student(int x) {
        id = x;
    }
};

Destructor

A destructor runs automatically when an object is destroyed.

الـ Destructor يعمل تلقائياً عند انتهاء الكائن أو تدميره.
class Student {
public:
    ~Student() {
        cout << "Object destroyed";
    }
};
Memorize:
  • Constructor: same name as class, no return type.
  • Destructor: ~ClassName(), no return type, no parameters.
  • Base constructor runs before derived constructor.
  • Derived destructor runs before base destructor.
احفظ: الـ Constructor نفس اسم الصنف ولا يملك نوع إرجاع. الـ Destructor يبدأ بـ ~. عند الوراثة، Constructor الأب يعمل أولاً، وDestructor الابن يعمل أولاً عند النهاية.

4. Static Class Data

English: A normal data member belongs to each object. A static data member is shared by all objects.

بالعربي: المتغير العادي لكل كائن نسخة خاصة منه. أما static فله نسخة واحدة مشتركة بين كل الكائنات.
#include <iostream>
using namespace std;

class Player {
private:
    static int count;

public:
    Player() {
        count++;
    }

    int getCount() {
        return count;
    }
};

int Player::count = 0;

int main() {
    Player p1, p2, p3;
    cout << p1.getCount();  // 3
}
Exam trap: You must define static data outside the class: 
int Player::count = 0;
خطأ امتحاني مهم: لا يكفي إعلان static داخل الصنف، يجب تعريفه خارج الصنف باستخدام ClassName::variable.

5. Function Overloading

English: Same function name, different parameter list.

بالعربي: نفس اسم الدالة، لكن معاملات مختلفة.
int sum(int a, int b) {
    return a + b;
}

double sum(double a, double b) {
    return a + b;
}

int sum(int a, int b, int c) {
    return a + b + c;
}
Can overload?ExampleReason
Yessum(int,int) and sum(double,double)Different type
Yessum(int,int) and sum(int,int,int)Different number
Yesfun(int,double) and fun(double,int)Different order
Noint fun(int) and double fun(int)Return type only is not enough
قاعدة ذهبية: التحميل الزائد لا يعتمد على نوع الإرجاع فقط. يجب أن تختلف المعاملات: النوع أو العدد أو الترتيب.

6. Operator Overloading

English: Operator overloading teaches an operator how to work with objects.

بالعربي: التحميل الزائد للمؤثرات يعني أن نعلّم مؤثر مثل + أو ++ كيف يعمل مع كائنات من صنف صنعناه.

Unary Operator: ++

class Counter {
private:
    int count;

public:
    Counter() {
        count = 0;
    }

    Counter(int c) {
        count = c;
    }

    Counter operator++() {
        ++count;
        return Counter(count);
    }

    int getCount() {
        return count;
    }
};

int main() {
    Counter c1, c2;
    c2 = ++c1;
    cout << c2.getCount();
}
If operator++ returns void, you can write ++c1; but not c2 = ++c1;.
إذا كانت دالة operator++ ترجع void فلا يمكن استخدامها في assignment مثل c2 = ++c1;.

Binary Operator: +

class Items {
private:
    int item_count;

public:
    Items() {
        item_count = 0;
    }

    Items(int c) {
        item_count = c;
    }

    Items operator+(Items other) {
        int total = item_count + other.item_count;
        return Items(total);
    }

    int getItem() {
        return item_count;
    }
};

int main() {
    Items panadol(5), aspirin(6), result;
    result = panadol + aspirin;
    cout << result.getItem();   // 11
}
Binary operator rule: In panadol + aspirin, panadol is the calling object, and aspirin is the parameter.
في panadol + aspirin الكائن الموجود على اليسار هو الذي يستدعي الدالة، والكائن على اليمين يرسل كمعامل.

Operators You Cannot Overload

.      member access
.*     pointer-to-member
::     scope resolution
?:     ternary conditional
sizeof size operator

7. Inheritance

English: Inheritance means using an existing class as a base to create a new class.

بالعربي: الوراثة تعني استخدام صنف موجود كأساس لإنشاء صنف جديد.

General Syntax

class Base {
    // base features
};

class Derived : public Base {
    // new features
};

Simple Example

#include <iostream>
using namespace std;

class Person {
private:
    int age;

public:
    void setAge(int a) {
        age = a;
    }

    void printAge() {
        cout << age << endl;
    }
};

class Student : public Person {
private:
    int level;

public:
    void setLevel(int l) {
        level = l;
    }

    void printLevel() {
        cout << level << endl;
    }
};

int main() {
    Student st;
    st.setAge(20);      // inherited public function
    st.setLevel(2);     // own function
    st.printAge();
    st.printLevel();
}

8. Polymorphism and Virtual Functions

English: Polymorphism means many forms. In C++, it appears mainly as overloading and virtual functions.

بالعربي: تعدد الأشكال يعني أن نفس الواجهة تظهر بأشكال متعددة. في C++ يظهر في Overloading و Virtual Functions.

Without virtual

class Base {
public:
    void print() {
        cout << "Base";
    }
};

class Derived : public Base {
public:
    void print() {
        cout << "Derived";
    }
};

int main() {
    Derived d;
    Base* p = &d;
    p->print();   // Base
}

With virtual

class Base {
public:
    virtual void print() {
        cout << "Base";
    }
};

class Derived : public Base {
public:
    void print() {
        cout << "Derived";
    }
};

int main() {
    Derived d;
    Base* p = &d;
    p->print();   // Derived
}
Simple idea: Without virtual, pointer type controls the function. With virtual, real object type controls the function.
ببساطة: بدون virtual نوع المؤشر يحدد الدالة. مع virtual نوع الكائن الحقيقي يحدد الدالة.

Pure Virtual Function and Abstract Class

class Shape {
public:
    virtual void draw() = 0;   // pure virtual
};

class Circle : public Shape {
public:
    void draw() {
        cout << "Circle";
    }
};
A class with at least one pure virtual function is abstract. You cannot create an object directly from it.
الصنف الذي يحتوي دالة pure virtual واحدة على الأقل يسمى abstract class ولا يمكن إنشاء كائن منه مباشرة.

Virtual Base Class

Used to solve ambiguity in multiple inheritance when the same base class appears twice.

تستخدم لحل الغموض عندما يرث صنف من طريقين يؤديان إلى نفس الصنف الأب.
class Parent {
public:
    int data;
};

class Child1 : virtual public Parent { };
class Child2 : virtual public Parent { };

class GrandChild : public Child1, public Child2 {
public:
    void setData() {
        data = 10;   // OK: only one Parent copy
    }
};

9. Friend Functions and Friend Classes

English: A friend function is not a member of the class, but it can access private/protected data.

بالعربي: دالة friend ليست عضواً في الصنف، لكنها تستطيع الوصول إلى بيانات private/protected.

Friend Function Example

class Point {
private:
    int x, y;

public:
    Point(int a, int b) {
        x = a;
        y = b;
    }

    friend int sum(Point p);
};

int sum(Point p) {
    return p.x + p.y;
}

Friend Between Two Classes

class B;   // forward declaration

class A {
private:
    int x;
public:
    A() { x = 10; }
    friend int add(A a, B b);
};

class B {
private:
    int y;
public:
    B() { y = 20; }
    friend int add(A a, B b);
};

int add(A a, B b) {
    return a.x + b.y;
}

Friend Class

class Rectangle {
private:
    int width, length;

public:
    Rectangle() {
        width = 3;
        length = 5;
    }

    friend class Square;
};

class Square {
public:
    int area(Rectangle r) {
        return r.length * r.length;
    }
};
Friend traps:
  • Friend function is called like normal function: sum(p), not p.sum().
  • friend keyword appears in declaration inside class, not in function definition.
  • Friendship is not mutual: if B is friend of A, A is not automatically friend of B.
  • Friendship is not inherited.
أخطاء friend: دالة friend تستدعى مثل دالة عادية وليس باستخدام النقطة. العلاقة ليست متبادلة وليست موروثة.

10. Templates

English: Templates let you write one function/class for many data types.

بالعربي: القوالب Templates تسمح بكتابة دالة أو صنف واحد يعمل مع أنواع بيانات مختلفة.

Function Template

template <class T>
T sum(T a, T b) {
    return a + b;
}

int main() {
    cout << sum(2, 3);
    cout << sum(2.5, 3.5);
}

Two Type Parameters

template <class T1, class T2>
T2 sum(T1 a, T2 b) {
    return a + b;
}
If parameters have different types, one T may not be enough. Use T1 and T2.
إذا كانت المعاملات من أنواع مختلفة، قد لا يكفي T واحد. استخدم T1 و T2.

Swap Template

template <typename T>
void func_swap(T &a, T &b) {
    T temp = a;
    a = b;
    b = temp;
}

Class Template

template <class T>
class MyClass {
private:
    T a, b;

public:
    MyClass(T x, T y) {
        a = x;
        b = y;
    }

    T getMax() {
        return (a > b ? a : b);
    }
};

int main() {
    MyClass<int> obj1(100, 75);
    MyClass<double> obj2(1.5, 1.3);
}
class and typename are usually interchangeable in template headers.
في تعريف القوالب غالباً يمكن استخدام class أو typename بنفس المعنى.

11. More Practice Questions

Quick MCQs

  1. Default access in class is: A public B private C protected D friend
  2. Constructor has: A return int B return void C no return type D return class
  3. Correct inheritance syntax: A class B : public A B class B extends A C B inherits A D class B(A)
  4. Runtime polymorphism needs: A static B virtual C friend D private
  5. Function overloading depends on: A return type only B parameter list C comments D variable names
  6. Friend function is: A always member B not member but can access private C constructor D destructor
  7. Pure virtual syntax is: A virtual void f(); B void f() = virtual; C virtual void f() = 0; D pure void f();
  8. Use arrow with: A object B pointer C class name D namespace
  9. Static data is shared by: A one object B all objects C no objects D only derived objects
  10. Cannot overload: A + B ++ C :: D ==
