Phase 2 — Next 90 Minutes: Deep Dive on Key Areas

Topic 1: Classes, Objects, Encapsulation, Constructors, Static

Explain Like I’m 5

A class is a cookie cutter. An object is the cookie. The class says what every cookie has. Each object gets its own values.

Private data is like putting your toys in a locked box. Other people cannot touch them directly. They must ask through public functions.

بالعربي ببساطة: الصنف مثل قالب البسكويت، والكائن هو البسكويتة نفسها. الصنف يحدد الشكل العام، وكل كائن له بياناته الخاصة. البيانات الخاصة private مثل صندوق مغلق؛ لا يمكن تعديلها مباشرة من خارج الصنف، بل عن طريق دوال عامة مثل setData() و getData().

Simple Code Example

#include <iostream>
#include <string>
using namespace std;

class Student {
private:
    int id;
    string name;

public:
    Student() {          // constructor
        id = 0;
        name = "none";
    }

    Student(int i, string n) {  // overloaded constructor
        id = i;
        name = n;
    }

    void setData(int i, string n) {
        id = i;
        name = n;
    }

    void getData() {
        cout << id << " " << name << endl;
    }

    ~Student() { }       // destructor
};

int main() {
    Student s1;
    Student s2(10, "Ali");
    s1.setData(1, "Sara");
    s1.getData();
    s2.getData();
}
Exam traps:
أخطاء امتحانية: أعضاء class في C++ تكون private افتراضياً. لا تستطيع الوصول إلى البيانات الخاصة مباشرة. الـ Constructor بلا نوع إرجاع، والـ Destructor يبدأ بعلامة ~.
  • In C++, class members are private by default. Struct members are public by default.
  • You cannot access s1.id if id is private.
  • Constructor name must be exactly the same as the class name and has no return type.
  • Destructor is ~ClassName(), has no return type, and usually no parameters.
  • static class data is shared by all objects and must be defined outside the class: int ClassName::x = 0;

Topic 2: Overloading, Operator Overloading, Templates

Explain Like I’m 5

Overloading means one name can do different jobs depending on what you give it. Like “open”: open a door, open a book, open a file.

Operator overloading means teaching C++ what +, ++, etc. mean for your own objects.

Templates are a magic mold: write one function/class that works for int, double, char, etc.

بالعربي ببساطة: التحميل الزائد يعني نفس الاسم لكن استخدام مختلف حسب المعطيات. مثال: دالة sum قد تجمع أعداد صحيحة أو أعداد عشرية. أما Operator Overloading فهو أن نعلّم C++ معنى المؤثر + أو ++ عندما يستخدم مع كائنات من صنف صنعناه. القوالب Templates تجعلنا نكتب كوداً عاماً يعمل مع أكثر من نوع بيانات.

Simple Code Example: Function Overloading

int sum(int a, int b) {
    return a + b;
}

double sum(double a, double b) {
    return a + b;
}

Simple Code Example: Operator Overloading

class Counter {
private:
    int count;

public:
    Counter() { count = 0; }
    Counter(int c) { count = c; }

    Counter operator++() {       // prefix ++obj
        ++count;
        return Counter(count);   // nameless temporary object
    }

    int getCount() {
        return count;
    }
};

Simple Code Example: Function Template

template <class T>
T sum(T a, T b) {
    return a + b;
}
Exam traps:
أخطاء امتحانية: لا يكفي اختلاف نوع الإرجاع لعمل Overloading. يجب اختلاف نوع أو عدد أو ترتيب المعاملات. إذا كان operator++ يرجع void فلا تكتب c2 = ++c1;.
  • Function overloading differs by type, number, or order of parameters. Return type alone is not enough.
  • void operator++() can do ++c1; but not c2 = ++c1; because it returns nothing.
  • Binary operator as member: left operand is the calling object, right operand is the parameter.
  • Some operators cannot be overloaded: ., .*, ::, ?:, sizeof.
  • Template with mixed types may need multiple template parameters: template<class T1, class T2>.

Topic 3: Inheritance, Virtual Functions, Friend Functions

Explain Like I’m 5

Inheritance means a child class gets things from a parent class. A Car is a kind of Vehicle.

Virtual functions are like saying: “Use the real object’s version, not just the pointer type’s version.”

A friend function/class is someone you trust with your private secrets.

بالعربي ببساطة: الوراثة تعني أن صنفاً ابناً يأخذ خصائص ودوال من صنف أب. مثال: Car يرث من Vehicle. الدالة الافتراضية virtual تجعل البرنامج يختار دالة الكائن الحقيقي وقت التشغيل، وليس فقط حسب نوع المؤشر. أما friend فهي دالة أو صنف نعطيه إذناً خاصاً للوصول إلى بيانات private/protected.

Simple Code Example: Inheritance

#include <iostream>
using namespace std;

class Vehicle {
public:
    Vehicle() {
        cout << "Base constructor\n";
    }

    void move() {
        cout << "Vehicle moves\n";
    }
};

class Car : public Vehicle {
public:
    Car() {
        cout << "Derived constructor\n";
    }

    void drive() {
        cout << "Car drives\n";
    }
};

int main() {
    Car c;
    c.move();   // inherited
    c.drive();  // own function
}

Simple Code Example: Runtime Polymorphism

class Base {
public:
    virtual void print() {
        cout << "Base\n";
    }
};

class Derived : public Base {
public:
    void print() {
        cout << "Derived\n";
    }
};

int main() {
    Derived d;
    Base* p = &d;
    p->print();   // Derived, because print is virtual
}

Simple Code Example: Friend Function

class Point {
private:
    int x, y;

public:
    Point(int a, int b) {
        x = a;
        y = b;
    }

    friend int sum(Point p);
};

int sum(Point p) {
    return p.x + p.y;   // allowed because sum is friend
}
Exam traps:
أخطاء امتحانية: Overloading يعني نفس الاسم بمعاملات مختلفة. Overriding يعني صنف ابن يعيد تعريف نفس الدالة الموجودة في الأب بنفس التوقيع. في C++ لا توجد كلمة interface مثل Java؛ نستخدم abstract class بدالة pure virtual.
  • Overloading vs overriding: overloading = same name, different parameters. Overriding = derived class redefines same prototype from base class.
  • C++ has no Java-style super or this for base calls. Use Base::function(). C++ does have this pointer to the current object.
  • Abstract class vs interface: C++ has abstract classes using pure virtual functions: virtual void draw() = 0;. There is no interface keyword in standard C++.
  • Private members of the base class are inherited but not directly accessible by derived class.
  • For runtime polymorphism, use base class pointer/reference and declare the function virtual in the base class.
  • Friend is not mutual and not inherited. If A declares B as friend, B can access A, but A does not automatically access B.

Extreme Summary Page 1: Definitions + Rules

  • Class: blueprint containing data + functions.
  • Object: instance of class.
  • Encapsulation: data + functions packed together.
  • Data hiding: private/protected restrict access.
  • Abstraction: show necessary behavior, hide internal details.
  • Inheritance: derived class gets base class members.
  • Polymorphism: one interface, many forms.
  • Constructor: initializes object, same name as class, no return type.
  • Destructor: cleanup function, ~ClassName(), no return type.
  • Static data: one shared copy for all objects.
  • Friend function: non-member function allowed to access private/protected members.
  • Friend class: all member functions of friend class can access private/protected members.
  • Function overloading: same function name, different parameter list.
  • Operator overloading: define operator behavior for class objects.
  • Template: generic code independent of data type.
بالعربي للحفظ: الصنف قالب، الكائن نسخة، التغليف جمع البيانات والدوال، إخفاء البيانات يمنع الوصول المباشر، الوراثة إعادة استخدام من الأب، تعدد الأشكال نفس الواجهة بأشكال مختلفة، والقالب Template كود عام لأكثر من نوع.

Extreme Summary Page 2: Signatures + Rules

class ClassName {
private:
    int x;
public:
    ClassName();
    ~ClassName();
};

ClassName::ClassName() { }
ClassName::~ClassName() { }

class Derived : public Base { };

class C : public A, public B { };

virtual void f();

virtual void draw() = 0;   // pure virtual

friend int sum(Point p);

template <class T>
T maxValue(T a, T b);

class Box { T value; };
Do not forget:
لا تنسَ: استخدم النقطة . مع الكائن، والسهم -> مع المؤشر. ضع virtual في الصنف الأب. واكتب فاصلة منقوطة بعد نهاية تعريف الصنف.
  • private cannot be accessed outside the class.
  • Derived class cannot directly access private base data.
  • Base constructor runs before derived constructor.
  • Destructor runs reverse order: derived then base.
  • Virtual function prototype must match in base and derived.
  • Use -> with pointers: ptr->func().
  • Use . with objects: obj.func().
