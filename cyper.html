Section 1: Multiple Choice Questions (MCQ)

30 questions Answers are hidden by default for active recall.

Q1. Which course aim best reflects an advanced threat-analysis focus?
A. Formatting secure documents
B. Analysis of APTs and zero-day attacks
C. Designing office layouts
D. Optimizing database indexes only
Correct answer: B. Lecture 1 frames APTs and zero-day attacks as a central course aim.
Q2. In the CNSS definition, information security primarily protects information and its critical elements, especially:
A. Speed, storage, and compression
B. Confidentiality, integrity, and availability
C. Marketing, finance, and operations
D. Hardware, firmware, and cables only
Correct answer: B. The CIA triad is the core set of critical information characteristics.
Q3. The McCumber Cube is another name for which model?
A. OWASP Top 10
B. NSTISSC/CNSS Security Model
C. SDLC Waterfall Model
D. STRIDE threat model
Correct answer: B. Lecture 2 identifies the NSTISSC/CNSS Security Model as the McCumber Cube.
Q4. Which statement best expresses the security-access trade-off?
A. Perfect security is always achievable with enough budget.
B. Strong security should eliminate all user access.
C. Security must balance protection with reasonable availability/access.
D. Availability should always override protection.
Correct answer: C. Lecture 2 emphasizes that perfect security is impossible and security must balance protection and access.
Q5. Which approach to security implementation is initiated by upper management?
A. Bottom-up approach
B. Top-down approach
C. Peer-to-peer approach
D. Ad hoc approach
Correct answer: B. The top-down approach begins with senior management direction and support.
Q6. Which of the following is NOT one of the four business functions of information security?
A. Protecting the organization's ability to function
B. Protecting data collected and used
C. Safeguarding technology assets
D. Guaranteeing unlimited access for all users
Correct answer: D. Information security enables safe access, but not unlimited access.
Q7. A threat is best defined as:
A. A weakness in a control
B. An object, person, or entity that represents constant danger to an asset
C. A successful exploit only
D. A completed incident report
Correct answer: B. Lecture 3 defines threat as a constant danger to an asset.
Q8. Which pair best distinguishes competitive intelligence from industrial espionage?
A. Legal vs. illegal collection
B. Encrypted vs. unencrypted traffic
C. Insider vs. outsider attack
D. Physical vs. logical storage
Correct answer: A. Competitive intelligence is legal; industrial espionage is illegal access or collection.
Q9. Shoulder surfing is primarily an example of:
A. Communications interception using packet capture
B. Unauthorized observation of confidential information
C. Energy anomaly
D. Software obsolescence
Correct answer: B. It occurs when someone observes confidential information while a user accesses it.
Q10. Which threat category includes fire, flood, earthquake, and lightning?
A. Information extortion
B. Forces of nature
C. Technical software failures
D. Compromise to intellectual property
Correct answer: B. Natural events disrupt storage, transmission, and use of information.
Q11. Hardware is the physical technology that:
A. Only authenticates users
B. Houses/executes software, stores/transports data, and provides input/output interfaces
C. Only controls passwords
D. Replaces people and procedures
Correct answer: B. Lecture 4 defines hardware through these three roles.
Q12. Which is NOT one of the seven major sources of physical loss listed in Lecture 4?
A. Liquids
B. Living organisms
C. Energy anomalies
D. SQL injection
Correct answer: D. SQL injection is a software/web application risk, not a physical loss source.
Q13. Tailgating occurs when:
A. A user changes passwords too often
B. An unauthorized person follows an authorized person through a control point
C. A firewall rejects legitimate packets
D. A worm changes its file size
Correct answer: B. It is a physical access-control bypass.
Q14. What is the key purpose of a mantrap?
A. Encrypt all web traffic
B. Prevent or control entry by using a two-door verification enclosure
C. Detect malware signatures
D. Increase wireless bandwidth
Correct answer: B. A mantrap verifies an individual before allowing entry into the protected facility.
Q15. Why is electronic monitoring considered incomplete as a standalone control?
A. It is always illegal.
B. It is usually reactive and recordings may not be monitored in real time.
C. It cannot record events.
D. It only works with biometric locks.
Correct answer: B. Lecture 4 stresses that monitoring records events but often does not prevent them.
Q16. Software is difficult to secure largely because:
A. It never changes after deployment.
B. Programming errors account for many information attacks.
C. It is not part of information systems.
D. It cannot be attacked remotely.
Correct answer: B. Lecture 5 identifies programming errors as a substantial source of attacks.
Q17. Trap doors in software are:
A. Purposeful shortcuts that can bypass security checks
B. Physical emergency exits only
C. Encrypted backup files
D. Anti-theft alarms
Correct answer: A. Trap doors are hidden or intentional software shortcuts that may cause breaches.
Q18. Which item belongs to the OWASP Top 10 list in Lecture 5?
A. Fire sprinklers
B. Broken authentication
C. Guards and dogs
D. Extreme temperature
Correct answer: B. Broken authentication is a listed web application risk.
Q19. Software Assurance (SA) means:
A. Adding security only after release
B. Building security into the development lifecycle
C. Removing testing from the SDLC
D. Replacing access control with user training only
Correct answer: B. SA integrates security into development rather than treating it as an afterthought.
Q20. Under the NIST approach, security must be considered during:
A. Disposal only
B. Implementation only
C. Initiation, development, implementation, operation, and disposal
D. Marketing and sales only
Correct answer: C. Lecture 5 lists these SDLC phases explicitly.
Q21. Which security design principle most directly expresses least privilege?
A. Programs/users utilize only necessary privileges.
B. Every user shares the same access level.
C. Protection mechanisms require no keys.
D. Minimize logging.
Correct answer: A. Least privilege means users and programs receive only the privileges needed.
Q22. For telework, which control is emphasized to secure data transmission?
A. VPN
B. Tailgating
C. Water detection
D. Phreaking
Correct answer: A. Lecture 5 states VPNs are essential for secure remote transmission.
Q23. Which sequence best distinguishes attack, vulnerability, and threat?
A. Potential danger; weakness; exploit action
B. Exploit action; weakness; potential danger
C. Weakness; potential danger; exploit action
D. Asset; control; policy
Correct answer: B. Attack = action exploiting weakness; vulnerability = weakness; threat = potential danger.
Q24. A virus differs from a worm because a virus:
A. Requires a host software environment to execute
B. Never replicates
C. Only attacks physical locks
D. Cannot be transmitted by email
Correct answer: A. Viruses attach to host programs; worms self-replicate without another host program.
Q25. Macro viruses activate when:
A. A user opens an infected document containing automatically executing macro code
B. A server loses water pressure
C. An ID badge is copied
D. A telephone network is phreaked
Correct answer: A. They are embedded in macro code used by office/database applications.
Q26. Which malware hides its true nature and releases a payload when activated?
A. Trojan horse
B. UPS
C. Badge
D. Mantrap
Correct answer: A. Trojan horses are disguised as useful software and may install back doors.
Q27. In DDoS attacks, compromised machines are commonly turned into:
A. Firewalls
B. Bots or zombies
C. Secure rooms
D. Badges
Correct answer: B. DDoS attacks coordinate traffic from many compromised hosts.
Q28. Pharming is best described as:
A. Sending bulk commercial email
B. Redirecting legitimate web traffic to illegitimate sites, often via DNS manipulation
C. Observing someone type a password
D. Executing a macro virus
Correct answer: B. Pharming redirects users to fake sites to collect personal information.
Q29. A sniffer is:
A. A program/device that monitors network data in transit
B. A lock category
C. A power supply device
D. A secure SDLC phase
Correct answer: A. Sniffers can be used legitimately for administration or maliciously to steal data.
Q30. TCP hijacking is a variant of:
A. Physical theft
B. Man-in-the-middle attack
C. Fire suppression
D. Competitive intelligence
Correct answer: B. The attacker inserts themselves into TCP/IP-based communication.

Section 2: True/False Questions

24 statements False statements include a correction.

T1. The course includes risk assessment, access control, IPS, and threat-landscape concepts.
True.
T2. Information security protects only computers, not people, procedures, or data.
False. It protects information and the systems, hardware, people, procedures, data, and networks that use it.
T3. The CIA triad consists of confidentiality, integrity, and availability.
True.
T4. The McCumber Cube is useful only when viewed from a single perspective.
False. A weakness emerges if it is viewed from only one perspective.
T5. Perfect security is impossible; the goal is a practical balance between protection and access.
True.
T6. The bottom-up approach is initiated by upper management.
False. Upper management initiates the top-down approach; bottom-up relies on individual administrators' technical expertise.
T7. Information security is both a management issue and a people issue.
True.
T8. Data at rest is important, but data in motion is not part of information security.
False. Protecting both data at rest and data in motion is critical.
T9. Employees can be among the greatest threats because mistakes can reveal, delete, modify, or improperly store data.
True.
T10. Competitive intelligence and industrial espionage are both illegal.
False. Competitive intelligence is legal; industrial espionage is illegal.
T11. Hardware security includes protecting the physical assets that house software, store data, and provide interfaces.
True.
T12. Unrestricted physical access can undermine the guarantee of information security.
True.
T13. ID cards and badges should be the only access control mechanism because they cannot be duplicated.
False. Cards can be duplicated, stolen, or modified; they should not be the only control.
T14. Electronic monitoring is often reactive rather than preventive.
True.
T15. UPS devices can keep systems online long enough for safe shutdown during power problems.
True.
T16. Software attacks and technical software failures are the two main software threat categories.
True.
T17. Bugs are code problems and unknown loopholes.
True.
T18. Software Assurance means security is added only after deployment.
False. SA builds security into the software development lifecycle.
T19. The NIST SDLC security approach considers security during operation and disposal as well as earlier phases.
True.
T20. Telework lowers risk because home networks are always more secure than office systems.
False. Telework increases risk; home devices and networks are often less secure, so VPNs and strong authentication are needed.
T21. A vulnerability is a weakness, while an attack is an action that exploits a weakness.
True.
T22. Worms require a host program to replicate.
False. Worms self-replicate without another host program; viruses require a host environment.
T23. Hoaxes can overload networks and waste users' time even if they are based on fake warnings.
True.
T24. Phishing and pharming are identical; both rely only on DNS manipulation.
False. Phishing deceives users by posing as a legitimate entity; pharming redirects legitimate traffic to illegitimate sites, often through DNS manipulation.

Section 4: Key Topics Summary

Lecture 1 - Introduction to Course

  • Course aims include APTs and zero-day attacks, risk assessment and mitigation, authentication, authorization, and accountability, and IPS implementation.
  • Expected outcomes: understand information security principles, assess risks, develop mitigation strategies, and implement cybersecurity protection techniques.
  • Course outline spans security fundamentals, security software, threats, protection techniques, risk management, firewalls, and current issues.

Lecture 2 - Fundamental Concepts of Information Security

  • Information security: a well-informed assurance that information risks and controls are in balance.
  • History: mainframe-era physical controls, Enigma, ARPANET, R-609, Internet growth, and the modern interdependence of connected networks.
  • CNSS definition: protection of information and critical elements including confidentiality, integrity, and availability, supported by policy, technology, training, and awareness.
  • CIA triangle: confidentiality, integrity, availability; expanded with timing, accuracy, authenticity, utility, and possession.
  • NSTISSC/CNSS Security Model (McCumber Cube): considers security goals, information states (storage, processing, transmission), and safeguards (policy, education/training, technology).
  • Information system components: software, hardware, data, people, procedures, and networks.
  • Security vs. access: perfect security is impossible; usable protection must preserve reasonable availability.
  • Top-down security is initiated by upper management; bottom-up relies on administrators' technical expertise.
  • SDLC provides a rigorous methodology to avoid missing steps and create a comprehensive security posture.

Lecture 3 - Objectives of Information Security and Threats

  • Primary mission: ensure systems and contents remain trustworthy despite daily attacks.
  • Four business functions: protect functionality, enable safe application operation, protect collected/used data, and safeguard technology assets.
  • Threat: an object, person, or entity representing constant danger to an asset.
  • Major threat categories include IP compromise, software attacks, quality-of-service deviations, espionage/trespass, forces of nature, human error, extortion, weak policy/planning/controls, sabotage/vandalism, theft, hardware/software failures, and technological obsolescence.
  • Intellectual property: ownership of ideas and control over their tangible or virtual representation; software piracy is a common breach.
  • Espionage/trespass: unauthorized access to protected information; distinguish legal competitive intelligence from illegal industrial espionage.
  • Human error includes non-malicious mistakes such as disclosure, incorrect entry, deletion/modification, unsafe storage, and failure to protect information.
  • Technological obsolescence: outdated infrastructure can produce unreliable or untrustworthy systems; managerial planning helps prevent it.

Lecture 4 - Hardware Security

  • Hardware houses and executes software, stores/transports data, and provides interfaces for information input/output.
  • Physical security policies treat hardware as a physical asset and protect it from harm or theft.
  • Unrestricted physical access usually means information security cannot be guaranteed.
  • Information stored on portable hardware is often more valuable than the device itself.
  • Seven physical loss sources: extreme temperature, gases, liquids, living organisms, projectiles, movement, and energy anomalies.
  • Physical security controls include walls/fences/gates, guards/dogs, ID cards/badges, locks/keys, electronic monitoring, alarms, computer rooms/wiring closets, and interior walls/doors.
  • Tailgating: an unauthorized person follows an authorized user through a control point.
  • Mantrap: a controlled enclosure with entry and exit points that verifies access before facility entry.
  • Electronic monitoring records events but is often reactive and not watched in real time.
  • Fire, water, supporting utilities, and facility system maintenance must be integrated into physical security and disaster recovery plans.

Lecture 5 - Software and Personal Computer Security

  • Software includes applications, operating systems, and command utilities, and is one of the six IS components.
  • Software is difficult to secure because programming errors cause a substantial portion of attacks.
  • Software threats: software attacks and technical software failures.
  • Malware: code designed to damage, destroy, or deny service; includes viruses, worms, and Trojan horses.
  • Trap doors are purposeful shortcuts that bypass checks; bugs are code problems and unknown loopholes.
  • OWASP Top 10 risks include injection, broken authentication, sensitive data exposure, XXE, broken access control, security misconfiguration, XSS, insecure deserialization, insufficient logging/monitoring, and insecure direct object references.
  • Software Assurance (SA): build security into the SDLC rather than adding it late; target is vulnerability-free, confidently deployable software.
  • NIST SDLC approach: consider security in initiation, development, implementation, operation, and disposal.
  • Design principles include simple design, permission not exclusion, complete mediation, two-key protection, least privilege, and usable protection mechanisms.
  • Mobile/portable security: tracking, remote wipe, disabling, physical protection, strong passwords, disabled auto-login, and backups.
  • Telework requires secure transmission, especially through VPNs, strong passwords, and authentication.

Lecture 6 - Threats and Malicious Software

  • Attack: action exploiting a weakness; vulnerability: weakness in controls; threat: potential danger that always exists.
  • Software attacks may be technical (malware, DoS) or human-based (social engineering, phishing).
  • Zero-day attack: uses malware or vulnerability not yet known/handled by defenses.
  • Hidden malware includes bots, spyware, and adware.
  • Virus: malicious code requiring a host software environment; can spread via physical media, email attachments, or networks.
  • Macro viruses use auto-executing macros; boot sector viruses target startup/boot files.
  • Worm: self-replicates without another host program and may exhaust memory, disk, and bandwidth; Morris worm (1988) is a major case.
  • Trojan horse: disguises itself as helpful software and reveals malicious behavior when activated, often releasing a back door.
  • Polymorphic worm: advanced malware using multiple attack methods and changing external characteristics.
  • DoS overwhelms a target with requests; DDoS launches coordinated streams from many compromised machines (bots/zombies).
  • Email attacks include spam, mail bombs, and phishing.
  • Communications interception attacks include sniffers, spoofing, pharming, man-in-the-middle, and TCP hijacking.
